Monkey Business for Serious Systems: Why Chaos Engineering Isn’t Crazy

Chaos engineering is a powerful methodology that empowers organizations to identify and address potential weaknesses in their systems before they lead to critical failures. By embracing chaos, teams can build more resilient and reliable systems, providing a better experience for end-users. The real-life examples mentioned in this blog and the outlined process flow should serve as a guide for organizations looking to implement effective chaos engineering management in their software development lifecycle. Embrace the chaos, test resilience, and fortify your systems for the challenges of the dynamic digital landscape. Let us delve into real-time chaos experiments and the outcomes we saw by intentionally introducing faults into our production environment. While tools such as Gremlin or Litmus can facilitate these experiments, we opted for manual execution initially to grasp the impact on our system’s health and the speed of recovery from such incidents.

API Testing in CI/CD with Mock Tools

Comprehensive API testing is critical for delivering reliable applications. This talk focuses on the integration of API testing within CI/CD using mock tools. With mock tools, development teams can decouple from live dependencies, ensuring tests are repeatable and isolated from external factors The talk will touch up on mock libraries for unit testing and then move on to the relevance and criticality of API mocking. The talk will cover basic API design best practices, why integration testing is critical, and how managing dependencies in integration testing is a challenge. The talk will introduce the API mocks concepts, and cover the tool landscapes including but not limited to mock server/mockon/postman/wiremock, etc. The talk will conclude with a demo of a basic CI CD pipeline with integration tests implemented using open source mock tools.
